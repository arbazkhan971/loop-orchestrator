# Todo App — built by the autonomous SME team

A small, **zero-dependency** todo app (Node.js `node:http` REST API + vanilla-JS UI + `node:test` tests) that was built by Loop Orchestrator's autonomous team — a Product Manager decomposed the brief, a Backend SME built the store and API, a Frontend SME built the UI, and a QA SME reviewed each diff before it merged to `main`.

This is a real artifact of `loop run`, not a hand-written sample. The store and server were generated by Claude SMEs working in isolated git worktrees and accepted by an independent critic.

## Run it

```bash
cd examples/todo-app
npm start            # serves http://localhost:3000
npm test             # 11 node:test tests (store + API)
```

Open http://localhost:3000 and add / complete / delete todos.

## What's inside

| File | Built by | What it is |
|------|----------|------------|
| `store.js` | Backend SME | In-memory todo store: `list / add / toggle / delete` |
| `server.js` | Backend SME | `node:http` server: `GET/POST /api/todos`, `POST /api/todos/:id/toggle`, `DELETE /api/todos/:id`, serves `index.html` |
| `index.html` | Frontend SME | Vanilla-JS UI talking to the API |
| `store.test.js`, `server.test.js` | QA / CT | `node:test` coverage of the store and the API |

## Reproduce it with loop

The `loop.config.yaml` and `brief.md` here are exactly what produced this app. From a fresh scaffold:

```bash
loop learn                     # train the team on the project
loop run "Build the todo app per the brief: a node:http JSON REST API \
  (list/add/toggle/delete) with an in-memory store, a static index.html \
  with vanilla JS, and node:test tests. Zero dependencies." --execute
loop monitor                   # watch the SME team on one screen
```

The team decomposes the goal onto a shared board, implements in parallel worktrees, and the QA critic reviews and merges each task to `main` — only accepting work whose tests pass.
